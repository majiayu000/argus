# Product Spec

## Linked Issue

GH-102

complexity: large

## 用户问题

PR #101 将 agent capability 扫描迁移为语法感知分析后，仍有三类相邻的
P1 绕过面：求值器中的下载执行字符串未进入有界解析、移动或复制 agent
配置时只检查目标路径、以及 shell 变量由环境变量与字面后缀拼接时丢失凭证
来源。它们会让真实高风险行为被降级或漏报，使用户误以为扫描结果低风险。

## 目标

- 恢复并结构化覆盖求值器字符串中的远程下载执行检测。
- 对移动/复制操作的源路径和目标路径使用统一的 agent 配置敏感性判定。
- 在不执行脚本、不猜测动态值的前提下保留 shell 赋值中的凭证路径来源。
- 为三类绕过各增加正例，并保留关键负例，防止以扩大误报换取召回。

## 非目标

- 不递归解释任意层级的动态代码或命令字符串。
- 不执行、下载或联网验证被扫描脚本中的内容。
- 不扩展新的凭证目录、agent 配置目录或网络客户端清单。
- 不改变已有 finding ID、severity、decision 或 JSON 输出结构。
- 不通过新增单个 callee 特判绕过现有语义化 fact/classifier 分层。

## Behavior Invariants

1. B-001 当受支持的 shell 求值器收到一个可静态取得的首个字符串参数，且
   该字符串在同一层包含“受支持网络客户端 → shell 解释器”的管道时，扫描
   必须产生现有 remote-execution finding，并得出与直接管道相同的阻断结论。
2. B-002 求值器字符串只允许一次有界、非递归解析；空管段、缺失参数、非
   字符串或无法静态取得的参数不得被猜测成远程下载执行。
3. B-003 不含远程下载执行管道的求值器内容仍可保留既有 execution
   capability，但不得被错误升级为 remote execution。
4. B-004 对 `mv` 与 `cp` 形状的命令，只要有效源路径或有效目标路径命中
   现有 agent 配置路径规则，就必须产生现有 agent-config-write finding；
   选项不得被当作路径端点。
5. B-005 `mv`/`cp` 两端均不命中 agent 配置路径时不得产生
   agent-config-write；路径的 raw 形式与可静态解析形式必须使用同一判定。
6. B-006 当 shell 赋值由一个可执行环境变量引用与有序字面路径片段组成，
   后续网络发送以文件读取语义引用该变量时，来源证据必须保留足够的引用、
   字面后缀与读取上下文，以便现有敏感路径规则识别凭证内容并产生
   secret-exfil finding；仅发送路径字符串不得等价为读取文件内容。
7. B-007 动态后缀不得被猜测；纯字面凭证字段名、被单引号抑制的变量引用，
   以及未进入网络发送的数据不得仅因文本相似而产生 secret-exfil。
8. B-008 三类修复必须复用 fact 形状、静态值和有界解析抽象；同类输入在
   Bash/Python/JavaScript/TypeScript 已支持的语法路径上不得依赖逐个新增
   callee 分支。
9. B-009 既有“独立网络调用 + 独立安全子进程”“writer payload 中仅出现
   配置路径文本”“非敏感 move/copy”负例必须继续保持非阻断。
10. B-010 任一输入无法安全解析时必须保守保留既有 capability 或
    analysis-incomplete 证据；不得静默伪造高置信结论，也不得吞掉扫描错误。

## 验收标准

- [ ] 求值器字符串回归测试证明有界的一层远程下载执行被阻断。
- [ ] `mv` 与 `cp` 的源端、目标端、raw/resolved 与非敏感矩阵均有测试。
- [ ] shell 变量引用加字面凭证路径后缀的链式网络发送被识别为 secret-exfil。
- [ ] 动态值、纯字面名称与无关 payload 负例不产生新增高风险 finding。
- [ ] `cargo test -p argus-agent` 与仓库确定性检查通过。

## 边界情况清单

| 类别 | 判定（covered: B-xxx / N/A + 原因） |
| --- | --- |
| 空/缺失输入 | covered: B-002 |
| 错误与失败路径 | covered: B-002, B-010 |
| 授权/权限 | N/A：本变更为离线静态扫描，不读取外部权限状态 |
| 并发/竞态 | N/A：单次扫描只消费内存中的不可变脚本内容 |
| 重试/幂等 | covered: B-008；同一输入重复扫描必须走同一结构化事实路径 |
| 非法状态转换 | N/A：不引入持久化状态机 |
| 兼容/迁移 | covered: B-003, B-005, B-009 |
| 降级/回退 | covered: B-002, B-010 |
| 证据与审计完整性 | covered: B-006, B-007, B-008 |
| 取消/中断 | N/A：扫描中断由现有调用方处理，本变更不新增可恢复状态 |

## 发布说明

这是向后兼容的检测正确性修复。用户可见变化仅限此前漏报或降级的三类输入
现在产生既有 finding；不新增 CLI 参数、配置迁移或输出字段。
